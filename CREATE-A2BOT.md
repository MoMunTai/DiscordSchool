# วิธีสร้าง A2Bot (และบอทตัวอื่นๆ) ให้ถูกต้อง

> จุดที่คนพลาดบ่อยที่สุด = สับสนระหว่าง **Application ID / Public Key / Token**
> สิ่งที่เอามาใส่ `.env` คือ **Bot Token** เท่านั้น (ขึ้นต้นด้วยตัวอักษรยาวๆ มี 2 จุด `.`)

## ขั้นตอน

### 1. สร้าง Application

1. ไปที่ https://discord.com/developers/applications
2. กดปุ่ม **New Application** (มุมขวาบน)
3. ตั้งชื่อ `A2Bot` → กด **Create**

### 2. เปิดใช้ Bot + เอา Token

1. เมนูซ้าย เลือก **Bot**
2. ดูตรงชื่อบอท — ใต้ชื่อมีปุ่ม **Reset Token** (บางบัญชีต้องกด "Add Bot" ก่อนถ้ายังไม่มี)
3. กด **Reset Token** → ยืนยัน → **Copy** token ที่ขึ้นมา
   - ⚠️ Token จะโชว์ครั้งเดียว! ถ้าปิดหน้าไปต้อง Reset ใหม่ (อันเก่าใช้ไม่ได้ทันที)
   - ⚠️ อย่าเอา **Application ID** หรือ **Public Key** มาใส่ — ผิดตัว
4. เอา token มาวางใน `.env`:
   ```
   A2_TOKEN=วาง_token_ตรงนี้
   ```

### 3. (สำคัญ) ปิด "Requires OAuth2 Code Grant"

- ในหน้า **Bot** เลื่อนหา **Requires OAuth2 Code Grant** → ต้อง **ปิด (OFF)**
- ถ้าเปิดไว้ บอทจะเชิญเข้าเซิร์ฟเวอร์ไม่ได้แบบลิงก์ปกติ

### 4. เชิญบอทเข้าเซิร์ฟเวอร์ (ด้วยสิทธิ์ที่ถูกต้อง)

1. เมนูซ้าย **OAuth2** → **OAuth2 URL Generator**
2. ในช่อง **SCOPES** ติ๊ก ✅ `bot`
3. ในช่อง **BOT PERMISSIONS** ที่โผล่มา ติ๊ก ✅ (เหมือนกันทุกบอท):
   - `View Channel`
   - `Connect`
   - `Speak`
   - (ไม่ต้องให้ Administrator ก็ได้ ขอแค่ 3 อันนี้พอ)
4. ก๊อปลิงก์ด้านล่างสุด → เปิดในเบราว์เซอร์ → เลือก **เซิร์ฟเวอร์เดียวกับบอทตัวอื่น** → Authorize

### 5. ตรวจให้แน่ใจว่าบอทเข้าห้องได้

- บอทต้องเห็นและเข้าห้องเสียง A2 ได้ — ถ้าห้องตั้ง permission เฉพาะ role
  ต้องเพิ่ม role ของ A2Bot หรือให้สิทธิ์ Connect/Speak ที่ห้องนั้นด้วย

## ✅ เช็คลิสต์กันพลาด

- [ ] ใช้ **Token** ไม่ใช่ Application ID / Public Key
- [ ] แต่ละบอท **คนละ token** (LeaderBot / A1Bot / A2Bot ห้ามใช้ token ซ้ำกัน)
- [ ] ทุกบอทอยู่ใน **เซิร์ฟเวอร์เดียวกัน**
- [ ] บอทมีสิทธิ์ **Connect + Speak** ในห้องของตัวเอง
- [ ] กรอก `channelId` ใน `config.js` ครบ (เอามาจาก Copy Channel ID)

> โค้ดมีตัวตรวจ token ซ้ำให้แล้ว — ถ้าเผลอใช้ token เดียวกัน 2 บอท มันจะเตือนตอนรัน
