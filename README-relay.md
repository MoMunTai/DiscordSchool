# ระบบ Leader Broadcast (กระจายเสียงอาจารย์ใหญ่)

อาจารย์ใหญ่พูดในห้อง **Leader Room** → ทุกห้องเรียน (A1, A2, ...) ได้ยินพร้อมกันแบบ realtime

## หลักการ
- บอท 1 ตัว = อยู่ได้ 1 ห้องเสียงต่อเซิร์ฟเวอร์ → จึงต้องมีบอทหลายตัว
- `LeaderBot` ฟังเสียงในห้อง Leader → แปลงเป็น PCM → ส่งผ่าน bus ในโปรเซส
- `A1Bot`, `A2Bot`, ... แต่ละตัวอยู่คนละห้อง คอยเล่นเสียงที่รับมา

## เตรียมก่อนรัน
1. **สร้างบอทให้ครบ** ใน https://discord.com/developers/applications
   - เริ่มเทสต้องมี 3 ตัว: LeaderBot, A1Bot, A2Bot
   - แต่ละตัวเปิด intent: `SERVER MEMBERS` ไม่ต้อง แต่ voice ใช้ได้เลย (ใช้แค่ Guilds + GuildVoiceStates)
   - เชิญทุกบอทเข้าเซิร์ฟเวอร์ (ต้องมีสิทธิ์ Connect + Speak)
2. **ใส่ token** ใน `.env` → `LEADER_TOKEN`, `A1_TOKEN`, `A2_TOKEN`
3. **ใส่ Channel ID** ใน `config.js`
   - เปิด Developer Mode (User Settings > Advanced) แล้วคลิกขวาห้องเสียง > Copy Channel ID

## รัน
```
npm start
```
แล้วเข้าไปพูดในห้อง Leader Room → จะได้ยินใน A1, A2

## ขยายเป็น A1-A8 / B1-B8
1. สร้างบอทเพิ่ม + ใส่ token ใน `.env` (เช่น `A3_TOKEN=...`)
2. เปิดบรรทัดที่ comment ไว้ใน `config.js`

## ข้อจำกัด/ข้อควรรู้
- ออกแบบสำหรับ "คนพูดทีละคน" (อาจารย์ใหญ่) — ถ้าหลายคนพูดพร้อมกันในห้อง Leader เสียงจะซ้อน ไม่ได้ผสมเสียง
- ดีเลย์ราว 0.2–0.5 วินาที (near-realtime)
- รัน 12+ บอทพร้อมกันใช้ CPU/เน็ตพอสมควร ถ้าห้องเยอะควรรันบนเครื่อง/VPS ที่แรงพอ
- บอททุกตัวควรอยู่เซิร์ฟเวอร์เดียวกัน
